Now the EULA reads "encourages you to install and use this Application
solely and exclusively for your personal use." and we're planning for
the public facing repo to only use non-protected (by trademark etc )
images.  I guess they can change that later if they choose so maybe

Using a distributed version control system like git makes this sort of issue less of a problem. If those managing the main repo choses to push problematic changes you 1) make them aware of the problem, 2) make your repo public and 3) pull any unencumbered contributions from the central repo.
