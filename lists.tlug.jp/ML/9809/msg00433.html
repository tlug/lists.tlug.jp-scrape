On Thu, 17 Sep 1998, Takeshi Yokokawa 横舌逅跂闕　健  wrote:

> Hi all,
> 
> I wrote a report using pLaTeX, which is an extension of LaTeX
> made by ASCII Co. to handle Japanese, on Debian at home.
> The document reads couple of eps files using a graphics package 
> and there was no problem on Debian to compile or even preview by xdvi.
> Today, I brought it to my office and tried to compile it on TurboLinux.
> Surprisingly, pLaTeX system on TL seems not to support the graphics 
> package! Since my machine has a Debian system, too, so I just copied 
> files from Debian to TL, but didn't work. The platex command didn't 
> find a graphics.sty. 

Where did you put the Debian files?

At the minimum latex should look in the same directory as your 
.tex file for the graphics.sty.  If you copy the graphics.sty file
from Debian to your TL system and just put it in the same directory
as the file that you are running pLatex on, that should work.
