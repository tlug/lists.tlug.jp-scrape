Hello,
I asked for an overview of the state of Japanese support for GNOME
and GTK on the JTLUG list.  Daisuke Sato gave me this response.
I've done a rough translation into English.  Since my Japanese is not 
that good, I've included his original Japanese text below.  If you can 
read Japanese, it is better to read the Japanese. 

>  sato@example.com
>  I've got gimp 1.0.0 running with gtk+ 1.0.4.  It's compiled to use
>  Japanese (--with-locale=ja_JP.ujis  --with-xinput=xfree).  When
>  used with things like testgtkrc, the font name is changed with fontset
>  and Japanese can be displayed.  I've verified that Japanese can be
>  inputted into an entry area.  However, input does not work in a
>  a text area.  
>  # this is the same situation explained by Chisaka-san at the
>    April meeting.
>
>  * Japanese can be displayed
>  * The cursor does not support double-bytes and it moves in the center
>  * an internal gdk routine issues a warning when input is done with IM
>    (because there's no real harm, we haven't messed with the source)
>
>  This is what I've verified so far.  I've used global source traces,
>  but have not been able to find the source of the problem.  
>
>  For practical use, I've verified that input works with the gimp 
>  text boxes.  
>
>
>  Regards,
>  Daisuke Sato(densuke@example.com)
>
>  P.S. Japanese is probably not possible in KDE since there is not
>     adequate multi-byte support in Qt.  At least, not soon.  



佐藤@東邦大です。

craigoda@example.com さんが書きました
> 最近、TLUGのMLでGTKとQTの日本語サポートの話題が出ました。 Yuditの
> unicode エディタの開発者と強力してGTKのlibraryの開発をやりましょう
> という話しが出ました。  詳しく相談しませんでした。 詳しく相談する
> 前にQTとGTKの日本語 サポートについて情報を集めようと思いました。 KDEとGNOMEの
> アプリケーションはどんな日本語サポートがありますか。 GTKに
> 日本語サポートをコンパイルしたらGNOMEのアプリケーションを入力と
> 表示させられますか。

gimp 1.0.0のリリースにあわせて、gtk+ 1.0.4をインストールしています。
日本語を使えるように舌逅跂闕して(--with-locale=ja_JP.ujis --with-xinput=xfree)作
舌逅跂闕し、testgtkrc等の使用するフォント名の部分をfontsetで置換したところ、表示
に関しては問題なく見えます。entryの部分での日本語の入力まではできることを
確認しました。しかしながら、textでの入力では入力できませんでした。
# 4月のミーティングの際に舌逅跂闕坂さんに言われたまんまです
ただし、

 * 日本語の表示はできる
 * カーソルが2バイト文字列への意識がないみたいで、まんなかを歩く
 * gdk内のルーチンがIMによる入力の際に警告(warning)を出す
   (実害はないので、ソースをいじって出なくしています)

というところを確認しています。
なんとかならないかと、globalを使ってソーストレースをしていますが、今のとこ
ろ、原因の追及ができていません。

応用としては、gimpのテキストボックスで入力ができることを確認してます。
