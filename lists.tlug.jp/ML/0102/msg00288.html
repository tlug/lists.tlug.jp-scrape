>>>>> "Garance" == Garance Paris <gparis@example.com> writes:

    >> What is the output of `grep canna /etc/services'?

    Garance> None whatsoever - is something missing there?

Yes.  There should be a line

canna           5680/tcp                        # cannaserver

in there.  su to root and add it (it doesn't matter where, but it's
probably in port order and it looks nicer if you keep it that way).
Maybe that will fix things, maybe not.  It's a good idea, though.

    Garance> Sorry if this is really stupid, but I thought the
    Garance> localhost was my computer itself?

Yes.  That is where you're running the cannaserver, right?

    Garance> Do you mean I should write "localhost:(the number in
    Garance> /etc/services)", if I had one at all?

Yes.

Akaneカスタマイズファイルとして "/usr/lib/canna/default.canna" を用います。
ローマ字かな変換テーブルは "/usr/lib/canna/dic/default.cbp" を用います。
      部首辞書 "bushu"              を指定しています。
  システム辞書 "keishiki"           を指定しています。
  システム辞書 "hojoswd"            を指定しています。
  システム辞書 "hojomwd"            を指定しています。
  システム辞書 "fuzokugo"           を指定しています。
  システム辞書 "iroha"              を指定しています。

The above are concerned with your user I/O and dictionary mappings,
not server communication, so they don't matter.

サーバ "unix" に舌逅跂闕します。

I think this is Canna's way of saying localhost.
