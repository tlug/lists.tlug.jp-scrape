>> The reason I didn't do this was because I assume 192.168.1.* adresses are
>> handled by the ISP,
>> and there is a chance that for example 192.168.1.11 is assigned to
>> somebody else, so if I set laptop or vmware to be the same IP it might
>> interfere with that.

No chance, unless the ISP is breaking the rules. 192.168.*.*
is one of a range of IP addresses that cannot be routed. They are
intended purely for private networks hiding behind NAT, portmapping,
etc.

This is all in RFC1597. The other such ranges are 10.0.0.0 -
10.255.255.255 and 172.16.0.0 - 172.31.255.255.

You can read all about this in some old lecture notes of mine:

http://www.csse.monash.edu.au/~jwb/subjects/cse5803/addr/addr_issues.html
