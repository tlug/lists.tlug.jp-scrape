[tlug] Trying to use ctrl-space instead of shift-space

  • Date: Wed, 6 Aug 2003 22:00:03 +0900
  • From: "Norman Diamond" <ndiamond@example.com>
  • Subject: [tlug] Trying to use ctrl-space instead of shift-space
I created file ~/.Xresources and inserted this line:
*ConversionStartKeys: Ctrl<Key>space

Great, now I can safely type a space even if I was still holding down
the shift key after the previous character.  Ctrl-space is a perfectly
fine way of firing up canna.

But ctrl-space doesn't turn off canna.  I still have to press the old
shift-space to turn off canna.

Anyone know how to make it really work?

By the way, I know that questions like this have been asked before on
the TLUG mailing list.  But when searching the archives, "shift" hits
no matches and "space" hits no matches.

(Following are two hints that I happened to find elsewhere on the net,
which helped me get as far as this point.  Maybe everyone else knows
these already, but I can get this additional copy into the archives
where they still won't be searchable :-)

=====

http://www.cam.hi-ho.ne.jp/oishi/tips.html#kinput2

Shift + Space による kinput2 起動の無効化最近の Linux では、インストール
したままの状態で利用してると Shift+Space と押しただけで canna や wnn が
(舌逅跂闕には kinput2 が) 起動する。私は SKK しか使わない (使えない?!) から
kinput2 は不要。ちょっとプログラムを書いてるときに ") {" と打とうとした
だけで kinput2 に起動されると迷惑なので Shift+Space による kinput2 の起動
を無効にすることに。

Shift+Space で kinput2 が起動しないようにするには
/usr/lib/X11/app-defaults/Kinput2
の
*ConversionStartKeys: Shift<Key>space
をコメントアウトする、つまり、冒頭に ! を付ければ良い。
# 舌逅跂闕ファイルの場所は、当然システムに依存する (と思う)

=====

http://www.linet.gr.jp/ML/plagia/19990831/3672.html

> これは .xinitrc あたりで、
> Kinput2*conversionStartKeys:\
>   Shift<Key>space \n\
>   <Key>Henkan
