Ryan Shaw <ryan.shaw@example.com> writes:

> M. Meiarashi wrote:
>
>> With Wnn, it depends on client.
>> For the case of kinput2, it further depends on the choice of ccdef.
>
> I am using kinput2. What is ccdef?

Maybe this helps:

    http://www.suse.de/~mfabian/suse-cjk/kinput2-wnn.html

If you use /usr/lib/X11/ccdef/ccdef.kinput2.egg, input typed in
hiragana will be converted to katakana with ``mod1-k'' (mod1 is
usually Alt):

	"n"	mod1-k		"ん"	to-katakana

Look into the file /usr/lib/X11/ccdef/ccdef.kinput2.egg to see all the
default key bindings.

For example, you can also switch to katakana before starting to
type by using

	"n"	mod1-control-k	"ん"	goto Katakana

the characters you input afterwards will be katakana immediately
and you don't need to convert them.
